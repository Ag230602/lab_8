1. Overview of My Individual Contribution
For Lab 8, I completed this work as an individual submission and took responsibility for the full domain adaptation workflow. 2. Specific Work Completed
My work included the following major tasks:

A. Domain Task Definition
I defined the domain task as uncertainty-aware disaster forecast explanation generation. In this task, the model receives forecast indicators such as probability of impact, confidence interval, rainfall intensity, surge level, exposure of vulnerable populations, and historical analog information. The expected output is a clear, decision-support explanation for emergency planners, response teams, or humanitarian stakeholders.

B. Instruction Dataset Creation
I created a domain-specific instruction dataset in JSON format. The dataset was designed to teach the model how to convert structured forecast signals into grounded, human-readable explanations. The examples include multiple scenario types such as:
- storm track uncertainty explanation
- flood response planning
- evacuation planning under uncertainty
- vulnerable population impact explanation
- resource allocation recommendations
- historical analog based explanation
- best-case and worst-case scenario summaries

The dataset was intentionally written in an instruction-input-output format so it could be used directly for supervised fine-tuning. I also ensured that the responses stayed close to the project goal of decision support rather than generic weather summarization.

C. Fine-Tuning Implementation
I implemented a LoRA-based fine-tuning workflow in Python using Hugging Face Transformers, PEFT, and TRL. I selected LoRA because it is efficient, practical for a course project, and appropriate when full model fine-tuning is too resource-intensive. The script supports:
- building the instruction dataset
- formatting the data for supervised fine-tuning
- applying LoRA adapters to the base language model
- training the model
- saving the adapted model
- evaluating baseline and adapted responses

D. Evaluation Setup
I created a baseline-versus-adapted evaluation structure with 10 representative disaster-related queries. These evaluation prompts test whether the adapted model is more domain-aware, more specific, and more useful for emergency decision support than the unadapted baseline model. The evaluation template stores:
- query id
- instruction
- input
- baseline response
- adapted response
- optional reviewer notes
- optional scoring fields

E. Planned System Integration
The adapted model is intended to be inserted into the existing project pipeline after retrieval. In the final integrated workflow:
User Query -> Retrieved Context / Forecast Evidence -> Adapted Model -> Decision-Support Explanation

This supports the larger architecture of my project, which combines forecasting outputs, uncertainty information, retrieved historical context, and human-readable decision support narratives.
